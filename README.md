# Energy of Fun: Hackathon Platform

## Overview
Собираем многостраничный сайт на Django, где главная «фишка» — RAG‑система на базе GigaChat, которая станет интеллектуальным помощником пользователя. Проект рассчитан на 48 часов непрерывной работы команды:

- **Федя** — тимлид и backend.
- **Родя** — фронтенд и UI/UX.
- **Сережа** — ML/AI и интеграция RAG.

## Repository Layout
- `backend/` — Django‑проект `core`, REST API, админка, интеграция с RAG.
- `frontend/` — клиентская часть (пока пусто, можно стартовать с Vite/React или Next.js).
- `ml/` — эксперименты с векторизацией, пайплайн RAG, адаптер для GigaChat.
- `docs/` — любая доп. документация, выписки из ТЗ, схемы.
- `.venv/` — локальное окружение Python (не коммитим).

## Local Setup
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt  # позже зафиксируем зависимости
cd backend && python manage.py runserver
```

## 48‑Hour Plan
| Время | Цели | Ответственные |
| --- | --- | --- |
| 0‑6ч | Разобрать ТЗ, описать данные, накидать CJM, финализировать UI‑киты | все |
| 6‑18ч | Django модели, базовые API, мок‑данные; прототип фронта; RAG‑Proof of Concept (пайплайн без GigaChat) | Федя / Родя / Сережа |
| 18‑30ч | Интеграция GigaChat, векторное хранилище (Qdrant/FAISS), авторизация | Сережа / Федя |
| 30‑42ч | UI‑полировка, связывание фронта с API, демо‑сценарии | Родя / Федя |
| 42‑48ч | Тесты, запись видео, репетиция питча, подготовка к сдаче | все |

## Task Board
- **Backend (Федя)**
  - Настроить Django‑проект, окружение, линтеры.
  - Реализовать модели, API, админку.
  - Подготовить endpoints для общения с RAG‑сервисом.
- **Frontend (Родя)**
  - Выбрать стек (Vite+React / Next), настроить дизайн‑систему.
  - Собрать основные страницы и состояния.
  - Подключиться к API + реализовать UX для помощника.
- **ML / AI (Сережа)**
  - Исследовать ТЗ, собрать базу знаний.
  - Настроить пайплайн: эмбеддинги → хранилище → GigaChat.
  - Обернуть RAG в сервис, готовый к вызовам из Django.
